### 流程图（DrawIt 纯文本图）

统一使用 [DrawIt](https://github.com/vim-scripts/DrawIt) Vim 插件画纯文本流程图，
不使用 Unicode box-drawing 字符。

**插件使用**：

- `\di` — 进入 DrawIt 模式
- `\ds` — 退出 DrawIt 模式
- 方向键画线（自动处理 `|` `-` `+` `\` `/`）

**字符约定**：

```
|     竖线
-     横线
+     转角 / T 形分支点
\     末分支转角（等价于 └）
/     上分支转角（等价于 ┌）
> 或 >  右箭头
< 或 <  左箭头
v 或 v  下箭头
^ 或 ^  上箭头
```

**常见模式**：

```
func()
    |
    +-> 分支1
    |
    \-> 末分支（退出 / 返回）

CPU-A              CPU-B
-----              -----
    |                  |
    v                  v
  step1             step1
```

**与 Unicode 字符的对应**（从旧文档迁移时参考）：

```
│ → |      ├ → +      └ → \      ─ → -      ┬ → +
▼ → v      ► → >      → → ->     ← → <-      ┴ → +
```

## 风格要点

### 语言

- 技术术语保留英文
- 中文描述配合英文术语
- 简短精确，不冗余
- 英文和中文之间不要有空格

### 换行

- 正文每行不超过80个字符
- 代码块、box-drawing图不受此限制

### 引用

- 函数/文件引用：`函数名()` 或 `file:line`
- 代码位置：`arch/arm64/kvm/mmu.c:123`

### 版本记录

```
-v0.1 2024.01.01 init
-v0.2 2024.01.02 增加xxx分析
-v0.3 2024.01.05 fix bug
```

## 段落间距规则

注意：
1. 简介和正文之间空两行
2. 其余各个section之间只空一行
3. 不要有"---"分割符
4. 各个段落之间也只空一行（标题和正文之间只空一行）
5. 所有括号使用英文括号

## 发布到 Hexo

笔记完成後，通过以下步骤转为 Hexo 博客文章：

1. 在 `/home/wz/hexo_blog_source` 下执行 `npx hexo new "文章标题"` 创建脚手架
2. 填写 frontmatter：
   - `description`: 引号包裹，内容为笔记的简介
   - `tags`: 相关技术标签
   - `categories`: 留空
3. 复制正文：去掉笔记的版本头(`-v0.x yyyy.mm.dd ...`)和标题行(Hexo 从 frontmatter 渲染标题)
4. 简介已放入 description，正文不再重复

注意：
- description 必须用双引号包裹，否则 YAML 语法错误
- 笔记是工作副本(有版本跟踪)，博客是发布版本(frontmatter 管理元信息)
