# 笔记写作 Skill

> 适用于技术文档、架构分析、学习笔记

---

## 文档结构

```
标题
===

-v0.1 yyyy.mm.dd Sherlock init
-v0.2 yyyy.mm.dd Sherlock xxx

简介：简要说明文档内容、目标、范围、作者
```

---

## 内容组织

### 层级结构

```
大标题 (标题用英文或中文)
---------

## 二级标题

### 三级标题

正文...
```

### 内容顺序

1. 概述/简介（说明文档目的）
2. 背景/前置知识
3. 核心逻辑/机制
4. 数据结构/函数
5. 调用流程
6. 注意事项
7. 参考文件

注意，不用严格是这样的顺序。

---

## 格式规范

### 代码块

````markdown
```c
// 语言标记的代码块
void func(void) {
    code;
}
```

```bash
# 命令行
./configure --enable-foo
```
````

### 表格

| 列1 | 列2 | 说明 |
|-----|-----|------|
| 值1 | 值2 | 描述 |

### 调用链/流程图

````markdown
```
函数A()                   // file:line
    │
    ├─► 函数B()
    │     └─► 子操作
    │
    ▼
函数C()
```
````

### 数据结构

````markdown
```c
struct foo {
    int field1;    // 字段说明
    uint64_t field2;
};
```
````

---

## 风格要点

### 语言

- 技术术语保留英文
- 中文描述配合英文术语
- 简短精确，不冗余

### 引用

- 函数/文件引用：`函数名()` 或 `file:line`
- 代码位置：`arch/arm64/kvm/mmu.c:123`

### 版本记录

````markdown
-v0.1 2024.01.01 init
-v0.2 2024.01.02 增加xxx分析
-v0.3 2024.01.05 fix bug
````

### 待办事项

````markdown
todo:

1. 待完成项1
2. 待完成项2
````

## 常见模式

### 状态机描述

```
状态A ───► 状态B
  │           │
  │           ▼
  └────◄──── 状态C
```

### 寄存器表格

| 偏移 | 名称 | 位域 | 描述 |
|------|------|------|------|
| 0x00 | CTRL | [31:0] | 控制寄存器 |

### 函数原型

```c
int kvm_xxx(struct kvm *kvm, u64 addr, u64 size);
// 参数说明
// 返回值说明
```
