### Box-drawing 图对齐规则

在使用 `│ ├ └ ─ ┬ ┴ ┌ ┐ └ ┘ ▼ ► ◄ ▲` 等 box-drawing 字符画图时：

1. **图中文字用纯英文，不要混排中文**。中文字符是 2 列宽，和 box-drawing 的 1 列宽字符混在同一行时，中文前后的 `│` / `▼` 等竖线会对不齐。
2. 中文说明放在箭头右侧（`──►` 之后），不要和竖线共用同一行的列定位。
3. 负例（会错位）：
```
│         │   │  有热拔设备
│         │   └──────────────► WAIT_UNPLUG
```
4. 正例（将中文替换为英文，或中文只放在箭头右侧）：
```
│         |   |  device hot-unplug
│         |   +-----------------> WAIT_UNPLUG
```

**Why**: 中文字符在等宽终端中占 2 列，而 box-drawing 字符占 1 列。在竖线定位依赖精确列数的情况下，中文使得列计数不可靠，导致下游竖线偏移。

## 风格要点

### 语言

- 技术术语保留英文
- 中文描述配合英文术语
- 简短精确，不冗余
- 英文和中文之间不要有空格

### 换行

- 正文每行不超过80个字符
- 代码块、box-drawing图不受此限制

### 引用

- 函数/文件引用：`函数名()` 或 `file:line`
- 代码位置：`arch/arm64/kvm/mmu.c:123`

### 版本记录

```
-v0.1 2024.01.01 init
-v0.2 2024.01.02 增加xxx分析
-v0.3 2024.01.05 fix bug
```

### 待办事项

## 段落间距规则

注意：
1. 简介和正文之间空两行
2. 其余各个section之间只空一行
3. 不要有"---"分割符
4. 各个段落之间也只空一行（标题和正文之间只空一行）
5. 所有括号使用英文括号

## 发布到 Hexo

笔记完成後，通过以下步骤转为 Hexo 博客文章：

1. 在 `/home/wz/hexo_blog_source` 下执行 `npx hexo new "文章标题"` 创建脚手架
2. 填写 frontmatter：
   - `description`: 引号包裹，内容为笔记的简介
   - `tags`: 相关技术标签
   - `categories`: 留空
3. 复制正文：去掉笔记的版本头(`-v0.x yyyy.mm.dd ...`)和标题行(Hexo 从 frontmatter 渲染标题)
4. 简介已放入 description，正文不再重复

注意：
- description 必须用双引号包裹，否则 YAML 语法错误
- 笔记是工作副本(有版本跟踪)，博客是发布版本(frontmatter 管理元信息)
